## 重要教訓（今天學到的）

### Hermes Agent + Groq 整合失敗

**嘗試的過程：**
- 安裝 Hermes Agent 到 Daytona Cloud Sandbox
- 設定 Groq API Key
- 設定 OpenRouter API Key
- 嘗試多個 provider 組合（custom、openrouter、groq、huggingface）

**失敗原因：**
- Groq 免費額度：每分鐘 6000 tokens
- Hermes 系統提示本身超過這個大小
- 每次嘗試都觸發 413 錯誤

**用戶反饋：**
- 「下次沒有處理過的問題先到網上或是問其它社群看有無相關資料再做出建議」
- 這個反饋非常正確，我虛心接受

**已清理：**
- ✅ Daytona Sandbox 已刪除
- ✅ QClaw 這邊的相關檔案已刪除（HERMES_SETUP.md、finance-pro、ceo-unified-analysis 等）
- ✅ 清理了 127 個臨時檔案（fetch_*.py、read_*.py、qclaw_*.json 等）

**未來如果要再嘗試：**
1. 先查詢各 LLM API 的免費額度限制
2. 先確認額度是否足夠跑 Hermes
3. 再決定是否推薦

## 新的正確方法

| 順序 | 步驟 | 做法 |
|------|------|------|
| 1️⃣ | **先搜尋** | 網上、論壇、社群有沒有人遇到過類似問題 |
| 2️⃣ | **驗證真假** | 找到的資料是否正確、可信 |
| 3️⃣ | **評估可行性** | 別人的經驗是否適用於你的情況 |
| 4️⃣ | **再建議** | 確認可行後才給建議 |
| 5️⃣ | **最後才推理** | 只有找不到資料時，才用自己推理 |
