# 全面警示系統技能

> 預警系統是投資的安全帶。

## 核心目的

投資最怕的不是虧損，而是沒有預警。這個技能提供多維度預警規則、風險等級分類、自動觸發機制，幫你及時發現風險。

---

## 警示系統框架

### 第一層：預警維度

| 維度 | 預警指標 | 觸發條件 | 風險等級 |
|------|---------|---------|---------|
| **基本面** | EPS、ROE、現金流 | 環比下降 > 20% | 🟡 中等 |
| **技術面** | 支撐位、RSI、MACD | 跌破支撐位 | 🟡 中等 |
| **法人動向** | 三大法人買賣超 | 連續 3 日齊賣 | 🟡 中等 |
| **總經環境** | PMI、失業率、CPI | PMI < 50 | 🟡 中等 |
| **產業週期** | 產業增長率、競爭格局 | 進入衰退期 | 🟡 中等 |
| **估值風險** | PE、PB、股價 | PE > 歷史平均 50% | 🟡 中等 |
| **流動性風險** | 成交量、股價波動 | 成交量萎縮 50% | 🟡 中等 |
| **財務風險** | 負債比率、利息覆蓋率 | 負債比率 > 70% | 🔴 高 |

### 第二層：風險等級分類

```
🟢 低風險（綠燈）：
- 基本面穩定或改善
- 技術面在支撐位上方
- 法人持續買超
- 總經環境良好
- 產業在成長期
- 估值合理
→ 可以加碼或持有

🟡 中風險（黃燈）：
- 基本面開始惡化
- 技術面接近支撐位
- 法人開始減碼
- 總經環境轉弱
- 產業進入成熟期
- 估值開始偏高
→ 應該減碼或觀望

🔴 高風險（紅燈）：
- 基本面明顯惡化
- 技術面跌破支撐位
- 法人連續齊賣
- 總經環境惡化
- 產業進入衰退期
- 估值明顯偏高
→ 應該停損或清倉
```

### 第三層：預警規則

#### 基本面預警
```
🟡 黃燈：
- EPS 環比下降 10-20%
- ROE 下降 1-2%
- 現金流比率下降 10-20%
- 毛利率下降 1-2%

🔴 紅燈：
- EPS 環比下降 > 20%
- ROE 下降 > 2%
- 現金流比率下降 > 20%
- 毛利率下降 > 2%
- 股息覆蓋率 < 1.0x
```

#### 技術面預警
```
🟡 黃燈：
- 股價接近支撐位（距離 < 5%）
- RSI 進入超買區（> 70）
- MACD 出現死叉信號
- 成交量開始萎縮

🔴 紅燈：
- 股價跌破支撐位
- RSI 極度超買（> 80）
- MACD 死叉確認
- 成交量大幅萎縮（< 50% 平均）
```

#### 法人動向預警
```
🟡 黃燈：
- 外資開始減碼（買賣超轉負）
- 投信開始減碼
- 累計買賣超從正轉負

🔴 紅燈：
- 三大法人連續 3 日齊賣
- 外資持股比例創新低
- 累計買賣超創新低
```

#### 總經預警
```
🟡 黃燈：
- PMI 從 > 50 跌到 50-52
- 失業率上升 0.5%
- CPI 上升 > 0.5%
- Fed 開始升息

🔴 紅燈：
- PMI < 50（景氣收縮）
- 失業率上升 > 1%
- CPI 上升 > 1%
- Fed 連續升息
```

#### 估值預警
```
🟡 黃燈：
- PE 高於歷史平均 30-50%
- PB 高於歷史平均 30-50%
- 股價高於 DCF 內在價值 20-30%

🔴 紅燈：
- PE 高於歷史平均 > 50%
- PB 高於歷史平均 > 50%
- 股價高於 DCF 內在價值 > 30%
```

### 第四層：自動觸發機制

```
系統自動監控以下指標：
1. 每日收盤後檢查技術面（支撐位、RSI、MACD）
2. 每週檢查法人動向（買賣超、持股比例）
3. 每月檢查基本面（EPS、ROE、現金流）
4. 每月檢查總經環境（PMI、失業率、CPI）
5. 每季檢查估值（PE、PB、DCF）

觸發條件：
- 任何一個維度達到黃燈 → 發出黃燈警示
- 任何一個維度達到紅燈 → 發出紅燈警示
- 多個維度同時達到黃燈 → 升級為紅燈警示
```

## 執行流程

### 步驟 1：定義預警規則
- 基本面預警規則
- 技術面預警規則
- 法人動向預警規則
- 總經預警規則
- 估值預警規則

### 步驟 2：設定監控指標
- 每日監控：技術面
- 每週監控：法人動向
- 每月監控：基本面、總經、估值

### 步驟 3：收集數據
- 自動爬取股票數據
- 自動爬取法人數據
- 自動爬取總經數據

### 步驟 4：計算指標
- 計算技術指標
- 計算基本面指標
- 計算估值指標

### 步驟 5：判斷風險等級
- 對比預警規則
- 判斷黃燈或紅燈
- 多維度綜合判斷

### 步驟 6：發出警示
- 黃燈警示：建議減碼或觀望
- 紅燈警示：建議停損或清倉

### 步驟 7：記錄和分析
- 記錄警示時間和原因
- 分析警示的準確性
- 優化預警規則

## 警示系統使用指南

### 黃燈警示（🟡 中風險）
**含義**：風險開始出現，需要警惕
**建議操作**：
- 減少新增投資
- 考慮部分減碼
- 加強監控

### 紅燈警示（🔴 高風險）
**含義**：風險明顯，需要立即行動
**建議操作**：
- 立即減碼或清倉
- 設置止損
- 等待風險解除

### 綠燈信號（🟢 低風險）
**含義**：風險可控，可以持有或加碼
**建議操作**：
- 可以持有
- 可以加碼
- 定期監控

## 常見誤區

❌ **誤區 1**：黃燈就一定要賣
✅ **正確**：黃燈是警示，要看具體情況。如果基本面還好，可以繼續持有。

❌ **誤區 2**：紅燈就一定要停損
✅ **正確**：紅燈是高風險，但不一定是虧損。要看止損位。

❌ **誤區 3**：綠燈就一定要加碼
✅ **正確**：綠燈是低風險，但不一定是好機會。要看估值。

❌ **誤區 4**：警示系統 100% 準確
✅ **正確**：警示系統只是參考，不是絕對。要結合自己的判斷。
