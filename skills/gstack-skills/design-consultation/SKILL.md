# Design Consultation - Design Partner

Design consultation skill for building design systems and facilitating design thinking.

## When to Use This Skill

Use this skill when the user says:

- "design consultation"
- "design system"
- "design workshop"
- "help me design this"
- Needs comprehensive design guidance

## Design Consultation Framework

### 1. Understand the Context

- What are you building?
- Who are the users?
- What's the brand/tone?
- What platforms (web/mobile/desktop)?

### 2. Design Principles

Establish guiding principles:

- **Clarity**: Every element has a purpose
- **Consistency**: Similar things look similar
- **Feedback**: System responds to user actions
- **Efficiency**: Common tasks are easy

### 3. Design System Components

Help define:

**Typography**
- Heading scale (H1-H6)
- Body text sizes
- Font families

**Colors**
- Primary palette
- Secondary palette
- Semantic colors (success/warning/error)
- Dark mode considerations

**Spacing**
- Base unit (4px or 8px grid)
- Spacing scale

**Components**
- Buttons (primary/secondary/ghost)
- Forms (inputs/selects/checkboxes)
- Navigation
- Cards/containers

### 4. Design Review

Review designs against:

- Brand guidelines
- Accessibility standards
- Platform conventions
- User mental models

## Output Format

```markdown
## Design Consultation

### Design Brief
[Summary of what's being designed]

### Design Principles
[3-5 guiding principles]

### Recommendations
[Specific design recommendations]

### Design System Tokens
[Colors, typography, spacing]

### Next Steps
[Action items for implementation]
```
