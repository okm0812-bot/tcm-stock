# Retro - Team Retrospective

Engineering manager skill for facilitating retrospectives and weekly reports.

## When to Use This Skill

Use this skill when the user says:

- "retro"
- "retrospective"
- "weekly report"
- "what went well"
- "sprint review"

## Retrospective Framework

### Classic Format: Start / Stop / Continue

**Start**: What should we start doing?
**Stop**: What should we stop doing?
**Continue**: What's working well that we should keep?

### Extended Format: 4Ls

**Liked**: What did you like?
**Learned**: What did you learn?
**Lacked**: What was missing?
**Longed For**: What do you wish you had?

## Weekly Report Template

```markdown
## Weekly Report - [Date]

### Accomplished This Week
- [Achievement 1]
- [Achievement 2]

### In Progress
- [Item 1] - [% complete]
- [Item 2] - [% complete]

### Blockers
- [Blocker 1] - Owner: [Name]

### Next Week Plan
- [Goal 1]
- [Goal 2]

### Metrics
- [Key metric 1]: [Value]
- [Key metric 2]: [Value]

### Team Health
[Brief note on team morale/energy]
```

## Retrospective Output

```markdown
## Sprint Retrospective - [Sprint/Date]

### What Went Well ✅
- [Item 1]
- [Item 2]

### What Could Be Better ⚠️
- [Item 1]
- [Item 2]

### Action Items 🎯
- [ ] [Action 1] - Owner: [Name] - Due: [Date]
- [ ] [Action 2] - Owner: [Name] - Due: [Date]

### Team Mood
[Emoji scale or brief description]
```
